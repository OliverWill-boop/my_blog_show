# 2026.3.29错题笔记（16届省赛+14届国赛）

```
range_show_mode = (++range_show_mode) % 3;

与

(++range_show_mode) % 3;

不同

前者会让range_show_mode在0到2循环，后者只会让range_show_mode递增
```

- 往往变量是对称的
- 先搭建框架再进行填写逻辑
- 为什么经常使用temp临时值？因为要把**数据获取**和**数据赋值**的两个逻辑分开，这样也方便后续调试

- 要注意处理边界情况，结束的时候要把应该清零的东西清零

```
if(seg_show_mode == 0)
			{
				case 4:  //单次测量触发按键
					range_show_mode = 1;
					signal_flag = 1;  // 单次测量标志

					summation_flag = 0;   //累加进程立即结束
					summation_index = 0;

					distance = 0;     //防止上一次测量结果影响当前测量
					distance_sum = 0;

					scheduler_task[4].last_ms = uwtick; // 立即执行Get_Distance任务
					Get_Distance();
				break;
				
				case 5:  //多次测量触发按键
					range_show_mode = 2;
					summation_flag = 1;
					signal_flag = 0;
					summation_index = 0;
					distance = 0;
					distance_sum = 0;
					// 根据选择的参数设置测量间隔时间
					scheduler_task[4].rate_ms = (unsigned int)para_time_arr[para_time_index] * 1000;
					scheduler_task[4].last_ms = uwtick;
					Get_Distance();
				break;
			}	
```

- 要注意这种**对于“立即执行”**的处理，还有**“防止上一次测量结果影响这次”**的处理
- **并且使用调度器的话，模块执行任务时间层面的逻辑，会和调度器有关**

```
void Get_Distance();
void Key_Proc();
void Led_Proc();
void Seg_Proc();
void Get_Temperature();
void DA_Proc();

typedef struct
{
	void (*task_func)(void);
	unsigned long int rate_ms; // 任务执行间隔，单位ms
	unsigned long int last_ms; // 上次执行时间
} task_t;


idata task_t scheduler_task[] = {
	{Led_Proc, 1, 0},
	{Seg_Proc, 20, 0},
	{Key_Proc, 10, 0},
	{Get_Temperature, 300, 0},
	{Get_Distance, 150, 0},
	{DA_Proc, 100, 0}
};

```

- 注意这一块调度器要在所有模块函数之前
