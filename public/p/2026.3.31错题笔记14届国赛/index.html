		问题：seg_buf[0] = 13; // F，这个没有正确显示，因为每一种情况，都需要seg_buf[0] = 13，所以我想着放前面就可以都作用到了。
		解析：
			这里的 seg_buf[0] = 12; // F 这行代码被放在了 switch 语句的大括号内，但在 case 标签之前。在 C 语言中， switch 				语句的执行流程是：

					1. 计算 switch 后的表达式值
					2. 直接跳转到匹配的 case 标签开始执行
					3. 跳过 case 标签之前的所有代码
			因此， seg_buf[0] = 12; 这行代码实际上不会被执行，因为程序会直接跳转到匹配的 case 标签（比如 case 0 ），跳过这行			代码。
         解决方案：把seg_buf[0] = 13放到switch前面

运算规则差异

  • 整数运算 ：

    • 除法：结果向下取整（如 5/2=2 ）
    • 取模：仅支持整数（如 5%2=1 ）
    • 无精度损失（只要不溢出）

  • 浮点数运算 ：

    • 除法：结果保留小数（如 5.0/2=2.5 ）
    • 取模：需使用 fmod() 函数（如 fmod(5.5, 2)=1.5 ）
    • 可能存在精度误差（如 0.1 + 0.2 != 0.3 ） 3. 类型转换

  • 隐式转换 ：

    • 整数与浮点数运算时，整数会自动转换为浮点数
    • 例如： int a=5; float b=2.0; a/b 结果为 2.5

  • 显式转换 ：

    • 浮点数转整数：直接截断小数部分（如 (int)3.9=3 ）
    • 整数转浮点数：无精度损失（如 (float)5=5.0 ） 4. 常见问题与解决方案

在规定时间内连续记录数据

  • 题目要求:

    • 在测距界面下，定义为距离“记录”按键，按下 S8 按键，开始记 录接下来 6 秒内距离数据的连续变化；数据记录过程中，所有按键动 作无效，直至数据记录完成后恢复。仅保留最近一次记录的一组数据。

  • 功能拆分：

    • 1.记录下六秒内数据的连续变化，并记录结束的时候保留最后一组数据

      • 需要一个数据存放数组和一个数组指针

      一定要注意数组的大小，是算出来的，就是在调度器任务清单上一般设置的200ms执行一次，这个不是要采集6s的数据吗，所以6000/200=30，数组的大小应该是30

      • 需要一个六秒的计时标志位
      • 需要一个记录标志位

    • 2.数据记录过程中，所有按键动作无效，直至数据记录完成后恢复

      • 对按键的作用域形成了一条规定

小tip:一般如果没有特殊要求，输入和输出就是对应的，就像这个题，题目没规定把采集的数据通过DAC输出，怎么输出。因为采集是每200ms执行一次（就是get_distance的周期），所以这里的输出也是200ms一次。

  • 下面这个写法会更好一些，因为一定要保持不同逻辑之间的清晰，和相互分离。按键检测，检测到按键按下是一个逻辑。是不是长按，又是另外一个逻辑。两个逻辑之间以long_func_flag这个标志位作为桥梁。另外满分写法还在最后写了一个清零的逻辑，防止上一次的结果影响这一次，这是一个很通用的 习惯
  • 上面那个写法，是我犯的一个错误，在硬件上会导致功能时灵时不灵的感觉，具体错误如下：

    • 重复执行问题

      • 当 time2s > 2000 时，只要按键仍然按下， sys_default() 会被重复执行
      • 每次进入循环都会检查时间并执行函数，可能导致系统状态不稳定

    • 状态管理混乱

      • 在执行 sys_default() 后立即重置 key_long_flag = 0
      • 这会导致后续的时间检查失效，因为标志位已经被清除

    • 按键松开处理缺失

      • 没有处理按键松开的情况，标志位可能保持为 1
      • 可能导致系统在按键松开后仍认为处于长按状态

    • 时间判断不准确

      • 时间检查仅在按键按下时执行，可能导致时间判断不连续
      • 如果按键按下时时间还没到 2 秒，后续即使时间到了也不会执行

    • 逻辑嵌套问题

      • 时间检查嵌套在按键检测内，导致逻辑依赖于按键的持续按下状态
      • 一旦按键状态有微小变化，可能导致整个逻辑流程被中断

{
	float da_temp;
	if(out_flag)//输出标志位拉高了
	{
		if(out_arr_index==30)//输出完毕
		{

		else//没输出完毕 则继续输出

			if(distance_arr[out_arr_index]<10)//距离小于10

				Da_Write(da_min_10x * 5.1);

			}
			else
			{
				da_temp = ((5.0 - (da_min_10x/10.0f))/80.0) * (distance_arr[out_arr_index] - 10) +(da_min_10x/10.0f); 
				Da_Write(da_temp * 51.0);
			}
		}		
	}

}

把十进制的数值以八位二进制的形式保存，并在led上展示出来

  • 把十进制数值转化为二进制的代码如下：
1
2
3
4

			for(i=0;i<8;i++)
				{
					ucled[i] = distance>>i;					
				}

  • 右移操作后， ucled[i] 会保留每一位的状态（0或1）
  • 例如，距离值80的二进制为 01010000 ，则：
  • ucled[0] = 80 » 0 = 80 (01010000) → 最低位为0
  • ucled[1] = 80 » 1 = 40 (00101000) → 次低位为0
  • ucled[2] = 80 » 2 = 20 (00010100) → 第三位为0
  • ucled[3] = 80 » 3 = 10 (00001010) → 第四位为0
  • ucled[4] = 80 » 4 = 5 (00000101) → 第五位为1
  • ucled[5] = 80 » 5 = 2 (00000010) → 第六位为0
  • ucled[6] = 80 » 6 = 1 (00000001) → 第七位为1
  • ucled[7] = 80 » 7 = 0 (00000000) → 最高位为0
